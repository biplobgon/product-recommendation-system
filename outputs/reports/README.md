# outputs/reports

CSV and text reports generated by the evaluation pipeline.

## Expected files

| File | Produced by | Description |
|------|-------------|-------------|
| `evaluation_report.csv` | `notebooks/04_model_evaluation.ipynb` | HR@K, NDCG@K, MRR@K, Precision@K, Recall@K, Coverage, Novelty per model per K |
| `eda_summary.txt` | `notebooks/01_eda.ipynb` | Key dataset statistics (row counts, unique items/visitors, date range) |
| `feature_summary.csv` | `notebooks/02_feature_engineering.ipynb` | Descriptive statistics for all engineered features |
| `model_training_log.txt` | `src/training/train.py` | Training loss/metrics per epoch, run timestamps |
| `hyperparameter_search.csv` | Optional grid/random search | Best hyperparameters per model and corresponding metric scores |

## evaluation_report.csv schema

```
model, k, hit_rate, ndcg, mrr, precision, recall, coverage, novelty
als, 10, 0.312, 0.187, 0.143, 0.031, 0.089, 0.54, 4.21
content_based, 10, ...
session_based, 10, ...
hybrid, 10, ...
```

## Interpreting results

- **Hit Rate@K**: fraction of test users for whom the ground-truth item appears in top-K recommendations
- **NDCG@K**: normalised discounted cumulative gain — rewards ranking the true item higher
- **MRR@K**: mean reciprocal rank of the first relevant item
- **Coverage**: fraction of the catalogue ever recommended (diversity proxy)
- **Novelty**: average log-popularity of recommended items (higher = more niche)
